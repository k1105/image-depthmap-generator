# Image → 3D Viewer (Depth Anything V2)

任意の写真・イラストを入力に取り、**AI で深度（デプス）を推定**して Three.js で立体化・プレビューするビューワーです。ビルド不要の単一 HTML（`index.html`）、サーバー側処理は無く、深度推定もブラウザ内で完結します。

## なぜデプスマップ方式か

- **ノーマルマップ**は各点の「傾き」しか持たず、高さの復元（積分）が不良設定問題のため、立体化すると必ず歪む。
- **デプスマップ**は各点の「高さ（奥行き）」そのものを持つので、`displacementMap` にそのまま渡すだけで歪みなく立体化できる。
- 写真からデプスマップを作るのが **Depth Anything V2**（[Transformers.js](https://huggingface.co/docs/transformers.js) でブラウザ内推論）。

## 使い方

`file://` だと CDN / モデル取得がブロックされる場合があるため、ローカルサーバー経由で開いてください。

```bash
cd image-3d-viewer
python3 -m http.server 8123
# → http://127.0.0.1:8123/index.html
```

- **画像の読み込み**: ドラッグ＆ドロップ / 「画像を開く」/ クリップボード貼り付け（Ctrl+V）
- **初回のみ** AI モデルをダウンロード（small で約 50MB、以後はブラウザにキャッシュ）
- WebGPU 対応ブラウザでは GPU 推論、非対応時は WASM にフォールバック
- **視点操作**: ドラッグで回転、ホイールでズーム、右ドラッグで平行移動

## アルファマスク（Nano Banana）

被写体だけを立体化し、背景を抜く機能。

1. ツールバーの **🔑 APIキー** に Gemini API キーを入力（localStorage に保存）
2. 画像を読み込むと、`gemini-3.1-flash-image`（Nano Banana 2）に「この画像のアルファマップを生成して」と送信
3. 返ってきたアルファマップで、**残った（白い）領域だけに depth を適用**し、抜かれた領域は変位させず非表示（`alphaTest`）
4. パネルの「アルファマスク」で ON/OFF・しきい値・再生成

> APIキーはブラウザ内（localStorage）に保存され、リクエストは `generativelanguage.googleapis.com` へ直接送られます。共有端末では使用後に空文字で上書きして消去してください。

## STL出力（型 / 3Dプリント）

レリーフを**直方体に彫り込んだ型（ネガ）**を、ミリ単位のバイナリ STL として書き出します。
出力メッシュは water-tight な多様体（非多様体エッジ 0、V−E+F=2 を検証済み）で、スライサーにそのまま読み込めます。

パネルの「STL出力（型）」で設定：

| 設定 | 説明 |
| --- | --- |
| 彫り込み高さ(mm) | レリーフ＝凹みの深さ（最も高い部分の彫り込み量） |
| 底マージン(mm) | 一番深い彫り込みの下に残す肉厚。総高さ = 彫り込み + マージン |
| 横幅(mm) | 直方体の footprint 幅（X）。縦（Y）は縦横比から自動 |
| 直方体の縦横比 | 画像に合わせる / 1:1 / 4:3 / 3:4 / 16:9 / 9:16。画像比と違う場合はレリーフを内側に収め、周囲は平らなリム |
| STL解像度 | グリッド分割（高いほど高精細・ファイル大） |

> 型なので形状は元のレリーフの**ネガ（鏡像の凹み）**です。背景・シルエットの縁はリム高さ（彫り込み 0）に揃います。

## コントロールパネル（右上）

| 項目 | 説明 |
| --- | --- |
| AIモデル | Depth Anything V2 の精度／速度（小・高速 / 中 / 大・高品質）。変更すると現在の画像で再推定 |
| 立体の深さ | 変位（displacement）の強さ |
| 奥行き反転 | 手前／奥が逆に飛び出すときに反転 |
| 写真を貼る | 表面に元画像を貼る（立体写真）／OFF で無地のレリーフ |
| ワイヤーフレーム | メッシュ表示 |
| メッシュ分割 | 平面の分割数（細かいほど高精細・重い） |
| 自動回転 | モデルを回す |
